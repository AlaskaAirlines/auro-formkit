        <auro-header level="2" id="a11yStates">Impact of State</auro-header>
        <auro-header level="3" id="a11yStateDisabled">Disabled</auro-header>
        <p><strong>What VoiceOver does:</strong></p>
        <ul>
          <li>Focuses the element and announces it (e.g., <em>"[label], dimmed, combo box"</em> on macOS / <em>"[label], dimmed"</em> on iOS)</li>
          <li>The word <strong>"dimmed"</strong> is VoiceOver's way of indicating disabled</li>
          <li>The element remains in the focus order (Tab still reaches it)</li>
          <li>The user cannot interact with it — the option list can not be expanded, and no selection can be made</li>
        </ul>
        <p>Key distinction from <code>aria-hidden</code>: A disabled element is <em>reachable but not operable</em>. If you want VoiceOver to skip it entirely, you would need <code>aria-hidden="true"</code> or <code>tabindex="-1"</code>, but that is generally not recommended for disabled controls — users benefit from knowing the field exists and why it may be unavailable.</p>
        <p>This is consistent across NVDA and JAWS as well, though the announcement wording differs (e.g., NVDA says <em>"unavailable"</em>).</p>
        <auro-header level="3" id="a11yStateMultiSelect">Multi Select</auro-header>
        <auro-header level="5" id="a11yStateMultiSelectLargeViewPort">Large Viewport VoiceOver Multi-select</auro-header>
        <p>Examples: large tablet, typical size or larger desktop browser window</p>
        <ol>
          <li><strong>Focus</strong></li>
          <strong>Open | Expand</strong>
          <li><strong>Navigate Options:</strong> Moves through options; announces each option name plus <em>"selected"</em> or nothing</li>
          <li><strong>Selecting an option:</strong> Double-tap toggles the focused option's selection — announces <em>"selected"</em> or <em>"unselected"</em></li>
          <li><strong>Close | Collapse:</strong> Collapses the list, announces the expanded state change
        </ol>
        <auro-header level="5" id="a11yStateMultiSelectSmallViewPort">Small Viewport VoiceOver Multi-select</auro-header>
        <p>Examples: mobile phone, very small desktop browser window</p>
        <ul>
          <li><strong>Focus</strong></li>
          <strong>Open | Expand</strong>
          <li><strong>Navigate Options:</strong> Moves through options; announces each option name plus <em>"selected"</em> or nothing</li>
          <li>Double-tap toggles the focused option's selection</li>
          <li><strong>Selecting an option:</strong> Double-tap toggles the focused option's selection — announces <em>"selected"</em> or <em>"unselected"</em></li>
          <li><strong>Close | Collapse:</strong> Collapses the list, announces the expanded state change
        </ul>

        <auro-header level="3" id="a11yStateInvalid">Invalid</auro-header>
        <p>When an <code>&lt;auro-select></code> is invalid the following occurs:</p>
        <ul>
          <li><code>Focus</code> — VoiceOver announces <em>"invalid data"</em> (macOS) or <em>"invalid entry"</em> (iOS) after the element's label</li>
          <li>The Help Text will render the error message and it will be included it in the VoiceOver focus announcement</li>
        </ul>
        <p>When focusing on an invalid <code>&lt;auro-select></code> the VoiceOver announces something like <em>"Country, invalid data, combo box, Please select a country"</em>.</p>
